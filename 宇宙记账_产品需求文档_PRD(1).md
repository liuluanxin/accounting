## 2. 产品概述

### 2.1 产品定位

「宇宙记账」是一款面向个人与家庭的**轻量级收支记账工具**。以"低门槛记录、可视化洞察、可控预算"为核心，帮助用户建立长期财务习惯。

### 2.2 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---|---|---|
| 学生 / 年轻职场人 | 收入有限、消费零散 | 快速记账、看清钱花哪了 |
| 家庭记账者 | 多人共享或家庭账本 | 多账户管理、月度预算控制 |
| 理财入门者 | 想了解资产负债 | 资产总览、分类统计 |

### 2.3 核心价值

1. **记一笔只需 3 秒**：首页数字键盘 + 分类快捷选择。
2. **看得懂的图表**：统计页环形图 / 柱状图直观呈现分类占比。
3. **花得明白**：预算页按分类设限，超支提醒。
4. **资产清晰**：资产页聚合多账户余额与净资产。

### 2.4 设计原则（宇宙梦幻风）

- **留白优先**：内容密度低，单屏聚焦一件事。
- **宇宙渐变配色**：浅蓝→淡紫整屏渐变，宇宙蓝（`#5B9BE0`）为主色点缀。
- **玻璃拟态卡片**：统一圆角、半透明渐变填充、轻投影区分层级。
- **一致性**：全局复用主题变量（见第 3 章），保证全站视觉统一。

### 3.5 固定状态栏与导航栏规范

**所有 12 个页面统一采用「固定状态栏 + 固定顶部导航栏」骨架**：状态栏与顶部导航栏始终固定在屏幕顶部，不随内容滚动而消失；内容区域在二者之下独立滚动。

**统一页面骨架**

```
┌────────────────────────────┐
│ 09:41  📶  🔋   固定状态栏    │  ← 浅白主题·深色文字·不滚动
├────────────────────────────┤
│ [←]   页面标题         [操作] │  ← 固定顶部导航栏 TopAppBar
├────────────────────────────┤
│                            │
│        页面内容（可滚动）      │
│                            │
├────────────────────────────┤
│ [首页][统计][资产][我的][日历]  │  ← 仅主 Tab 页固定显示底部导航
└────────────────────────────┘
```

**规则**
- **状态栏**：浅色背景 + 深色文字（或随主题反色）；内容区顶部设置安全内边距（safe area inset），避免内容被状态栏遮挡。
- **顶部导航栏**：固定置顶，含页面标题；二级页 / 认证页左侧提供「← 返回」，右侧按需放操作图标（如日历入口、周期/维度筛选、设置）。
- **底部导航栏（硬性约束）**：仅首页 / 统计 / 资产 / 我的 / 日历 5 个主 Tab 页固定显示；**二级页面（全部账单、预算、记账、分类管理、首页自定义）与认证页（登录 / 注册 / 忘记密码）一律不加底部 Tab 栏**，仅保留顶部导航栏。
- **滚动行为**：仅内容区滚动，状态栏与顶部导航栏始终可见。

### 3.6 图标库选型

> **结论：采用 Lucide（线性描边）作为全局图标库。**
> 理由：线性描边风格与记账页分类网格、顶部操作图标视觉统一；图标量达 1500+，完整覆盖 餐饮 / 交通 / 购物 / 社交 / 工资 / 红包 等全部业务分类（含餐具、车辆、飞机等食物与交通工具图标），无需自定义兜底；SVG 源码可直接内联，离线可用、无 CDN 依赖，颜色用 `currentColor` 跟随双主题变量。

> **双图标体系（全站统一）**：全局采用「线性 Lucide（操作 / 导航 / 账户图标）＋ 彩色 Twemoji（分类 / 账单图标）」双体系。分类图标（餐饮 / 交通 / 购物 / 社交 / 工资 / 红包 等）统一使用彩色 Twemoji SVG，在记账页、全部账单页、分类管理页、统计页分类行、日历页选中日账单明细、预算页分类预算中保持一致的渲染与配色；其余操作、导航、账户图标保持线性 Lucide，避免与分类图标混淆。

| 图标库 | 风格 | 图标量 | 授权 | 与 Tailwind 搭配 | 适配度 |
|---|---|---|---|---|---|
| **Lucide** ✅ 选定 | 线性描边（Feather 衍生） | 1500+ | ISC | 良好 | ★★★★★ |
| Heroicons (Outline) | 线性描边、圆润 | ~230（Outline + Solid） | MIT | 官方出品，原生契合 | ★★★★☆ |
| Material Symbols (Outlined) | 线性描边、几何感 | 3000+ | Apache-2.0 | 需手动对齐间距 | ★★★★☆ |
| Ant Design Icons | 线性 + 填充双风格 | 800+ | MIT | 需引入 Ant 生态 | ★★★☆☆ |
| Feather Icons | 极简线性描边 | ~290 | MIT | 良好 | ★★★★☆ |

**选型要点**
1. **风格统一**：操作 / 导航 / 账户图标必须线性描边（stroke），禁止混用 emoji；**分类与账单图标统一使用彩色 Twemoji**，与线性图标区分，提升分类识别度，且在全站各页面保持一致。
2. **可内联**：优先选用提供 SVG 源码的库，便于离线内联与单色主题（`currentColor`）。
3. **分类覆盖**：分类管理页（5.11）需覆盖 餐饮 / 交通 / 购物 / 社交 / 工资 / 红包 等常见分类图标；Lucide 已内置全部所需分类图标，无需自定义 SVG 补充。
4. **主题色跟随**：图标颜色统一用 `currentColor`，随主题 CSS 变量（cosmic / galaxy 双主题）切换，避免硬编码颜色。
5. **统一规格**：图标按钮圆形（999dp），栅格对齐；stroke-width 统一 1.5–2，保证粗细一致。

| 入口 | 类型 | 进入方式 |

| 日历 | 主 Tab | 底部主导航切换 |
| 预算页 | 二级页 | 首页预算卡片 / 资产页「本月预算」卡片 / 我的页「预算管理」进入 |

| 数据导入页 | 二级页 | 我的页「数据导入」进入 |

- 下拉刷新：首页、统计、资产、日历列表。

### 5.1 登录页面

**页面定位**：用户身份核验入口，未登录态的首屏。

**布局**

```
┌────────────────────────────┐
│                            │
│         🪐 宇宙记账          │  ← Logo + 产品名
│      让每一笔都清晰           │
│                            │
│  [ QQ邮箱 输入框      ]      │
│  [ 密码 输入框      👁 ]      │
│                            │
│   [    登 录  （主按钮）]     │
│                            │
│   ☑ 记住我   忘记密码？       │
│                            │
│        还没有账号？注册        │
└────────────────────────────┘
```

**核心功能**
1. QQ邮箱 + 密码登录（首期）。
2. 第三方登录（微信 / Apple / Google）二期开发，首期不接入，仅支持 QQ邮箱账号体系。
3. 「记住我」本地令牌续期。
4. 表单校验：格式、必填、密码可见切换。

**交互细节**
- 输入格式错误实时红框提示，不提交后才报错。
- 登录中按钮置灰 + 加载圈，禁止重复点击。
- 登录失败：Snackbar 提示原因（账号或密码错误 / 网络异常）。
- 已登录用户再次打开 App 跳过本页，直达首页。

**关键字段**

| 字段 | 类型 | 校验 |
|---|---|---|
| account | string | QQ邮箱格式（xxx@qq.com） |
| password | string | 8–20 位，含字母 + 数字 |
| rememberMe | bool | 默认 false |

**页面状态**
- 正常态：可输入、可提交。
- 加载态：按钮 loading。
- 错误态：校验失败 / 登录失败提示。
- 空态：无（首屏必有内容）。

### 5.2 注册页面

**页面定位**：新用户创建账号，登录页下游。

**布局**

```
┌────────────────────────────┐
│  ← 返回         注册         │
│                            │
│  [ QQ邮箱 输入框   获取验证码 ]│
│  [ 验证码 输入框 ]           │
│  [ 设置密码 输入框   👁 ]     │
│  [ 确认密码 输入框   👁 ]     │
│                            │
│   ☑ 我已阅读并同意《用户协议》│
│                            │
│   [    注 册  （主按钮）]     │
│                            │
│        已有账号？登录         │
└────────────────────────────┘
```

**核心功能**
1. QQ邮箱 + 邮箱验证码注册（首期）；手机号 + 短信验证码注册二期。
2. 密码强度校验（弱 / 中 / 强 实时提示）。
3. 协议勾选前置，未勾选禁用注册按钮。
4. 注册成功自动登录并进入首页。

**交互细节**
- 「获取验证码」点击后 60s 倒计时，期间禁用。
- 两次密码不一致实时提示。
- 验证码错误 / 过期给出明确文案。
- 协议未勾选时按钮置灰并 Toast 引导。

**关键字段**

| 字段 | 类型 | 校验 |
|---|---|---|
| email | string | QQ邮箱格式（xxx@qq.com） |
| emailCode | string | 6 位数字邮箱验证码 |
| password | string | 强度 ≥ 中 |
| confirmPwd | string | 与 password 一致 |
| agreeTerms | bool | 必须 true |

  │
  ▼ 通过
[设置密码] ──► 实时强度检测（弱 / 中 / 强）
[确认密码] ──► 与 password 比对
  ├─ 不一致 ──► 实时提示「两次输入的密码不一致」
  │
  ▼ 一致
[勾选《用户协议》]
  ├─ 未勾选 ──► 注册按钮置灰 + Toast 引导
  │
  ▼ 已勾选且全部通过
[点击「注册」]
  ├─ 失败 ──► Toast 错误原因 ──► 回到表单
  │
  ▼ 成功
自动登录 ──► 进入首页
```

#### 字段校验规则表

| 字段 | 校验时机 | 规则 | 错误提示 |
|---|---|---|---|
| email | 失焦 / 点获取验证码 / 提交 | 非空；须为 QQ 邮箱域名（`*@qq.com`）；长度 ≤ 64 | 「请输入正确的QQ邮箱」 |
| emailCode | 失焦 / 提交 | 非空；6 位纯数字；与后端下发一致且在有效期（5 分钟）内 | 「验证码错误」/「验证码已过期」 |
| password | 输入时实时 | 8–20 位；含字母 + 数字；强度 ≥ 中（中=字母+数字，强=字母+数字+特殊符号） | 实时强度标签；弱时提示「密码强度过低」 |
| confirmPwd | 输入时实时 / 提交 | 非空；与 password 完全一致 | 「两次输入的密码不一致」 |
| agreeTerms | 提交时 | 必须 true | 按钮置灰 + Toast「请先阅读并同意用户协议」 |

**页面状态**：正常 / 加载（发送验证码、注册请求）/ 校验错误（红框 + 提示）/ 协议未同意（按钮禁用）/ 注册成功（跳转首页）。

**页面定位**：新增资产账户的表单页，从资产页「+ 添加账户」进入；二级页，无底部 Tab 栏。

**布局**

```
┌────────────────────────────┐
│  ← 返回     创建账户        │  ← 固定顶部导航栏
├────────────────────────────┤
│  ┌──────────────────────┐   │
│  │ [账户图标] 招商银行储蓄卡  │   │  ← 实时预览卡片
│  │ ¥8,888.88             │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ 🏷 账户名称            │   │
│  │ 💰 初始余额            │   │
│  │ 选择账户图标           │   │
│  │ [💼][💳][💵][🟢微信][🔵支]│  ← 4 个通用图标 + 支付品牌
│  │ [🐧][📈][工][建][农]   │  ← 支付/投资 + 银行品牌
│  │ [中][交][招][浦][广]   │  ← 银行品牌 logo（特征字）
│  │ [民][信][JD][美团][花呗]│  ← 银行品牌 + 消费平台品牌

│  │ 重置                  │   │
│  └──────────────────────┘   │
└────────────────────────────┘
```

> 注：通用账户（钱包/银行卡/现金/投资）采用线性 Lucide 图标；支付/消费平台（微信/支付宝/QQ/京东/美团/花呗）使用品牌色圆形 + 简化图形 logo；银行类账户采用「品牌色圆形 + 特征字」方案（工/建/农/中/交/招/浦/广/民/信）。全站账户图标统一通过 `ic2()` 渲染，保持资产页、记账页、首页账户列表视觉一致。

**核心功能**
1. 新建账户：名称、图标、初始余额。
2. 实时预览：顶部卡片随输入即时更新名称与余额（图标随类型切换为 Lucide 或品牌 logo）。
3. 表单校验：账户名称为空时禁用「创建账户」按钮；空名称提交时 Toast 拦截。
4. 草稿重置：重新进入页面时清空草稿，避免上次残留数据。

**交互细节**
- 顶部导航栏固定，左侧「← 返回」回到资产页。
- 账户名称输入框：最大 20 字，输入时同步刷新预览卡名称；空时预览显示默认「新账户」。
- 初始余额输入框：支持小数，输入时同步刷新预览卡余额，默认 0.00；删除后余额归零。
- 账户图标网格：20 个预设项（4 通用 + 6 支付平台 + 10 银行品牌），区分通用图标与品牌 logo：
  - 通用图标（Lucide）：`wallet`（钱包）、`credit-card`（银行卡）、`banknote`（现金）、`trending-up`（投资）。
  - 支付 / 消费平台品牌（品牌色圆形 + 简化图形）：`wechat`（微信·绿）、`alipay`（支付宝·蓝）、`qq`（QQ·黑）、`jd`（京东·红）、`meituan`（美团·黄）、`huabei`（花呗·蓝）。
  - 银行品牌（品牌色圆形 + 特征字）：`icbc`（工·红）、`ccb`（建·蓝）、`abc`（农·绿）、`boc`（中·红）、`comm`（交·蓝）、`cmb`（招·红）、`spdb`（浦·深蓝）、`cgb`（广·红）、`cmbc`（民·绿）、`citic`（信·红）。
  - 默认选中 wallet；点击切换选中态与预览图标（品牌类展示对应品牌 logo）。
- 创建账户：校验通过后追加到账户列表，Toast「已创建：xxx」，然后返回资产页并刷新列表（资产页账户列表同样通过 `ic2()` 渲染品牌 logo）。
- 重置：清空名称、余额归零、图标恢复默认 wallet。

**关键字段**

| 字段 | 类型 | 说明 |
|---|---|---|
| accDraft.name | string | 账户名称，必填，trim 后非空 |
| accDraft.ic | string | 图标键：通用 `wallet` / `credit-card` / `banknote` / `trending-up`，或支付品牌 `wechat` / `alipay` / `qq` / `jd` / `meituan` / `huabei`，或银行品牌 `icbc` / `ccb` / `abc` / `boc` / `comm` / `cmb` / `spdb` / `cgb` / `cmbc` / `citic` |
| accDraft.bal | string | 初始余额字符串，提交时转 Number |
| isValid | bool | name.trim() !== ''，控制按钮禁用态 |

**页面状态**：正常编辑态 / 校验错误态（空名称提示）/ 创建成功态（返回资产页）。

**事件表**

| 事件 | 触发 | 行为 |
|---|---|---|
| 输入账户名称 | 用户输入 | state.accDraft.name = value；更新预览名称与按钮可用态 |
| 输入初始余额 | 用户输入 | state.accDraft.bal = value；更新预览余额 |
| 选择图标 | 点击图标格子 | state.accDraft.ic = key；更新选中态与预览图标 |
| 创建账户 | 点击主按钮 | 校验 → push 到 ACCOUNTS → Toast → 返回资产页 |
| 重置 | 点击重置 | 恢复 accDraft 默认值并重新渲染 |
| 进入页面 | 导航到 createAcc | 重置 accDraft 为默认值 |

### 5.15 数据导入页面

**页面定位**：从第三方平台批量导入历史账单，降低新用户迁移成本；由我的页「数据导入」入口进入，二级页，无底部 Tab 栏，仅保留顶部导航栏（标题「数据导入」+ 左侧「← 返回」我的页）。

**布局**

```
┌────────────────────────────┐
│  ←      数据导入             │  ← 固定顶部导航栏（返回 + 标题）
│  ┌──────────────────────┐   │
│  │ 📤 从其他平台导入账单   │   │  ← 顶部说明卡：支持 CSV/Excel/OFX，
│  │   支持常见格式，自动识别 │   │     自动识别收支与分类
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ 🏦 银行      各大银行   │   │  ← 来源卡 1：银行
│  │ [工][建][农][中][交][招]│   │     展示支持银行品牌 logo
│  │ [选择文件]  [开始导入]  │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ 🟦 支付宝   账单下载   │   │  ← 来源卡 2：支付宝（品牌 logo）
│  │ [选择文件]  [开始导入]  │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ 🟢 微信     微信支付   │   │  ← 来源卡 3：微信（品牌 logo）
│  │ [选择文件]  [开始导入]  │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ 📋 其他记账软件 随手记…│   │  ← 来源卡 4：其他记账软件
│  │ [随手记][鲨鱼][网易有钱]│   │     展示支持 App 标签
│  │ [选择文件]  [开始导入]  │   │
│  └──────────────────────┘   │
│  ┌──────────────────────┐   │
│  │ ℹ️ 导入说明 …          │   │  ← 底部说明卡
│  └──────────────────────┘   │
└────────────────────────────┘
```

**核心功能**
1. 顶部说明卡：概括导入能力（支持 CSV / Excel / OFX 等常见格式，自动识别收支与分类）。
2. 四类来源卡（按用户要求）：
   - **银行**：展示支持银行（工 / 建 / 农 / 中 / 交 / 招等）品牌 logo，支持各大银行流水导出文件（.csv / .xls）。
   - **支付宝**：品牌 logo，支持支付宝「账单下载」导出的 Excel 文件。
   - **微信**：品牌 logo，支持微信支付账单（.csv）。
   - **其他记账软件**：展示支持 App 标签（随手记 / 鲨鱼记账 / 网易有钱 / Timi 时光记账 / 叨叨记账等）。
3. 每张来源卡含「选择文件」（`<input type=file>`，accept `.csv,.xls,.xlsx,.ofx`）与「开始导入」按钮。
4. 底部导入说明卡：导入前建议先备份、重复账单自动合并、本地完成不上传。

**交互细节**
- 点击「选择文件」调起系统文件选择器，选定后 Toast 提示「已选择『文件名』，点击『开始导入』解析」。
- 点击「开始导入」：先 Toast「正在解析 XX 数据…」，约 1.2s 后 Toast「导入成功：已合并 28 条账单到宇宙记账」（原型阶段为模拟流程，后续接入真实解析引擎）。
- 解析引擎按 交易时间 / 金额 / 分类 自动匹配，重复账单合并；分类映射至既有分类体系（餐饮 / 交通 / 购物 / 娱乐 / 住房 / 工资 / 红包等，见 5.11）。
- 页面沿用全站宇宙梦幻风（浅蓝→淡紫整屏渐变、玻璃拟态卡片、星球装饰层），与主色宇宙蓝一致。

**支持来源与格式**

| 来源 | 入口标识 | 支持格式 | 说明 |
|---|---|---|---|
| 银行 | 🏦 + 品牌 logo | .csv / .xls | 工 / 建 / 农 / 中 / 交 / 招等主流银行流水 |
| 支付宝 | 🟦 支（品牌 logo） | .xls / .xlsx | 支付宝「账单下载」导出 |
| 微信 | 🟢（品牌 logo） | .csv | 微信支付账单 |
| 其他记账软件 | 📋 | .csv / .xls / .xlsx | 随手记 / 鲨鱼记账 / 网易有钱 / Timi 时光记账 / 叨叨记账等 |

**页面状态**：空态（未选文件）/ 已选文件态（等待导入）/ 解析中态 / 导入成功态（Toast 反馈）。

**事件表**

| 事件 | 触发 | 行为 |
|---|---|---|
| 选择文件 | 点击「选择文件」并选定 | 读取文件名；Toast 提示已选择，等待导入 |
| 开始导入 | 点击「开始导入」 | Toast「正在解析…」→ 延迟后 Toast「导入成功：已合并 N 条账单」 |
| 返回 | 点击顶部「←」 | 返回我的页（5.8） |

**导入后各画面联动（事件驱动）**

导入并非只停留在导入页：解析成功后，数据会合并进各画面共用的共享数据源，并派发 `dataImported` 事件；所有数据画面在 `render()` 时重新读取数据源，从而自动反映导入内容。

| 共享数据源 | 写入内容 | 联动画面 |
|---|---|---|
| `STAT_BILLS`（含时间戳账单） | 按来源追加 2 条示例账单（银行：日常-1280 / 工资+8500；支付宝：购物-368 / 餐饮-88；微信：社交-200 / 餐饮-56；其他：交通-120 / 零食-45） | 统计页（5.5）：收支总览合计、分类环形图、汇总表按新数据重算 |
| `BILLS`（按日期分组账单） | 追加到「今天」分组（items + sum） | 全部账单页（5.13）；首页近 7 天账单列表随之增长 |
| `ACCOUNTS`（账户列表） | 按来源新增账户（银行→导入-招商银行；支付宝→导入-支付宝；微信→导入-微信；其他→导入-其他账本），去重避免重复 | 资产页（5.6）、首页「我的账户」横向滚动 |
| `CAL`（日历日金额） | 选中日（7/12）累加导入金额（如银行：+7220） | 日历页（5.7）：选中日金额与点点标记同步更新 |

事件触发规则：
- 导入成功后调用 `applyImport(src)` 写入上述数据源，并 `dispatchEvent(new CustomEvent('dataImported', {detail:{src, count}}))`。
- 全局监听 `dataImported`：若当前停留在数据画面（首页 / 统计 / 资产 / 日历 / 全部账单），立即 `render()` 刷新当前屏；导入页自身则在导入后重渲染以展示「本次已成功导入 N 条」进度提示。
- 用户从其他入口（如「我的 → 数据导入」返回后）再进入任意数据画面时，因各画面均从共享数据源实时取数，天然呈现导入后的最新结果——无需逐页手动刷新。

> 原型阶段 `applyImport` 写入的是按来源预设的示例账单，用于演示「导入即联动」；正式实现将替换为真实文件解析引擎（CSV/Excel/OFX → 字段映射 → 分类匹配 → 去重合并）。
